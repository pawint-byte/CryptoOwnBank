## 1. Architecture Overview

```
User Flow:
  /signup → POST /api/auth/signup → Create user (unverified) → Send verification email → "Check Your Email" screen
  (user clicks email link) → GET /api/auth/verify-email/:token → email_verified = true
  /login → POST /api/auth/login → (check email_verified) → Create session → Redirect to app

Password Reset:
  /forgot-password → POST /api/auth/forgot-password → Send reset email
  /reset-password/:token → POST /api/auth/reset-password → Update password

Admin Migration (for users on legacy auth):
  Admin clicks "Migrate" → POST /api/admin/migrate-auth → Send reset email first, then update auth_provider → User sets password via reset link
```

**Key Design Decisions:**
- Users are NOT auto-logged-in after signup. They must verify their email first.
- Login is blocked for unverified accounts (returns `EMAIL_NOT_VERIFIED` code).
- Login returns `NO_ACCOUNT` code for unknown emails, so the frontend can suggest signing up.
- Password hashing uses Node.js built-in `scrypt` (no external dependency like bcrypt).
- Tokens are 32 random bytes, hex-encoded.
- Sessions use `express-session` with `connect-pg-simple` (PostgreSQL-backed).

```typescript
  emailVerifyToken: varchar("email_verify_token"),
  passwordResetToken: varchar("password_reset_token"),
  passwordResetExpires: timestamp("password_reset_expires"),
  authProvider: varchar("auth_provider", { length: 20 }).default("email").notNull(),
  isAdmin: boolean("is_admin").default(false).notNull(),

export type User = typeof users.$inferSelect;
```

**Key columns for auth:**
| Column | Type | Purpose |
|--------|------|---------|
| `password_hash` | varchar, nullable | Stores `salt:derivedKey` (scrypt) |
| `email_verified` | boolean, default false | Must be true before login is allowed |
| `email_verify_token` | varchar, nullable | Random token sent in verification email |
| `password_reset_token` | varchar, nullable | Random token sent in reset email |
| `password_reset_expires` | timestamp, nullable | Token expiry (1 hour for user-initiated, 72 hours for admin migration) |
| `auth_provider` | varchar(20), default "email" | Distinguishes auth method. Values: "email", "replit" (or other legacy) |
| `is_admin` | boolean, default false | Controls access to admin routes |

**For FamilyRoots.family:** Set `authProvider` default to `"email"` since there's no legacy auth to support. You can remove the `authProvider` column entirely if you'll never have multiple auth methods.

## 3. Auth Service

### `server/services/email-auth.ts`

```typescript
import { randomBytes, scrypt, timingSafeEqual } from "crypto";
import { promisify } from "util";

const scryptAsync = promisify(scrypt);

export async function hashPassword(password: string): Promise<string> {
  const salt = randomBytes(16).toString("hex");
  const derivedKey = (await scryptAsync(password, salt, 64)) as Buffer;
  return `${salt}:${derivedKey.toString("hex")}`;
}

export async function verifyPassword(password: string, hash: string): Promise<boolean> {
  const [salt, key] = hash.split(":");
  const derivedKey = (await scryptAsync(password, salt, 64)) as Buffer;
  const keyBuffer = Buffer.from(key, "hex");
  return timingSafeEqual(derivedKey, keyBuffer);
}

export function generateToken(): string {
  return randomBytes(32).toString("hex");
}

export function validateEmail(email: string): boolean {
  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return re.test(email) && email.length <= 254;
}

export function validatePassword(password: string): { valid: boolean; message: string } {
  if (password.length < 8) {
    return { valid: false, message: "Password must be at least 8 characters long" };
  }
  if (password.length > 128) {
    return { valid: false, message: "Password must be less than 128 characters" };
  }
  if (!/[A-Z]/.test(password)) {
    return { valid: false, message: "Password must contain at least one uppercase letter" };
  }
  if (!/[a-z]/.test(password)) {
    return { valid: false, message: "Password must contain at least one lowercase letter" };
  }
  if (!/[0-9]/.test(password)) {
    return { valid: false, message: "Password must contain at least one number" };
  }
  return { valid: true, message: "" };
}
```

**Why scrypt over bcrypt:**
- Built into Node.js `crypto` module — no native binary dependency
- Same security level, timing-safe comparison
- Portable across environments without compilation issues
- Storage format: `hex_salt:hex_derived_key` (e.g., `a1b2c3...:d4e5f6...`)

## 4. API Routes

All routes go in your main routes file. Here are the complete implementations:

### POST /api/auth/signup

```typescript
app.post("/api/auth/signup", async (req, res) => {
  try {
    const { email, password, firstName, lastName } = req.body;
    const { validateEmail, validatePassword, hashPassword, generateToken } = await import("./services/email-auth");

    if (!email || !password || !firstName) {
      return res.status(400).json({ message: "Email, password, and first name are required" });
    }

    if (!validateEmail(email)) {
      return res.status(400).json({ message: "Invalid email address" });
    }

    const passwordCheck = validatePassword(password);
    if (!passwordCheck.valid) {
      return res.status(400).json({ message: passwordCheck.message });
    }

    const { pool } = await import("./db");
    const existing = await pool.query("SELECT id FROM users WHERE LOWER(email) = LOWER($1)", [email]);
    if (existing.rows.length > 0) {
      return res.status(409).json({ message: "An account with this email already exists" });
    }

    const hashedPassword = await hashPassword(password);
    const verifyToken = generateToken();
    const { randomUUID } = await import("crypto");
    const userId = randomUUID();

    await pool.query(
      `INSERT INTO users (id, email, first_name, last_name, password_hash, email_verified, email_verify_token, auth_provider, created_at, updated_at)
       VALUES ($1, $2, $3, $4, $5, false, $6, 'email', NOW(), NOW())`,
      [userId, email.toLowerCase(), firstName, lastName || null, hashedPassword, verifyToken]
    );

    const appUrl = `${req.protocol}://${req.hostname}`;
    const verifyUrl = `${appUrl}/verify-email/${verifyToken}`;

    // Send verification email (adapt to your email service)
    try {
      await emailService.sendEmailVerification(email, firstName, verifyUrl);
    } catch (emailErr) {
      console.error("Failed to send verification email:", emailErr);
    }

    // Notify admin of new signup
    try {
      await emailService.sendAdminNewMember(ADMIN_EMAIL, `${firstName} ${lastName || ""}`.trim(), email);
    } catch (emailErr) {
      console.error("Failed to send admin notification:", emailErr);
    }

    // DO NOT auto-login — require email verification first
    res.status(201).json({
      message: "Account created! Please check your email to verify your address before logging in.",
      userId,
      requiresVerification: true
    });
  } catch (error) {
    console.error("Signup error:", error);
    res.status(500).json({ message: "Failed to create account" });
  }
});
```

### POST /api/auth/login

```typescript
app.post("/api/auth/login", async (req, res) => {
  try {
    const { email, password } = req.body;

    if (!email || !password) {
      return res.status(400).json({ message: "Email and password are required" });
    }

    const { pool } = await import("./db");
    const result = await pool.query(
      "SELECT id, email, first_name, last_name, password_hash, auth_provider, email_verified FROM users WHERE LOWER(email) = LOWER($1)",
      [email]
    );

    if (result.rows.length === 0) {
      return res.status(401).json({
        message: "No account found with that email. Would you like to sign up?",
        code: "NO_ACCOUNT"
      });
    }

    const dbUser = result.rows[0];

    // Block legacy auth users from email login
    if (dbUser.auth_provider !== "email" || !dbUser.password_hash) {
      return res.status(401).json({
        message: "This account uses a different login method."
      });
    }

    const { verifyPassword } = await import("./services/email-auth");
    const valid = await verifyPassword(password, dbUser.password_hash);
    if (!valid) {
      return res.status(401).json({ message: "Invalid email or password" });
    }

    // Block unverified accounts
    if (!dbUser.email_verified) {
      return res.status(403).json({
        message: "Please verify your email before logging in. Check your inbox for the verification link.",
        code: "EMAIL_NOT_VERIFIED"
      });
    }

    // Create session via Passport
    const user = { claims: { sub: dbUser.id }, authProvider: "email" };
    req.login(user, (err) => {
      if (err) {
        console.error("Login failed:", err);
        return res.status(500).json({ message: "Login failed" });
      }
      res.json({ message: "Logged in successfully" });
    });
  } catch (error) {
    console.error("Login error:", error);
    res.status(500).json({ message: "Login failed" });
  }
});
```

### POST /api/auth/forgot-password

```typescript
app.post("/api/auth/forgot-password", async (req, res) => {
  try {
    const { email } = req.body;
    if (!email) {
      return res.status(400).json({ message: "Email is required" });
    }

    const { pool } = await import("./db");
    const result = await pool.query(
      "SELECT id, first_name, auth_provider FROM users WHERE LOWER(email) = LOWER($1)",
      [email]
    );

    // Don't reveal whether the account exists
    if (result.rows.length === 0 || result.rows[0].auth_provider !== "email") {
      return res.json({ message: "If an account exists with that email, a reset link has been sent." });
    }

    const { generateToken } = await import("./services/email-auth");
    const resetToken = generateToken();
    const expires = new Date(Date.now() + 60 * 60 * 1000); // 1 hour

    await pool.query(
      "UPDATE users SET password_reset_token = $1, password_reset_expires = $2, updated_at = NOW() WHERE id = $3",
      [resetToken, expires, result.rows[0].id]
    );

    const appUrl = `${req.protocol}://${req.hostname}`;
    const resetUrl = `${appUrl}/reset-password/${resetToken}`;

    try {
      await emailService.sendPasswordReset(email, result.rows[0].first_name || "there", resetUrl);
    } catch (emailErr) {
      console.error("Failed to send reset email:", emailErr);
    }

    res.json({ message: "If an account exists with that email, a reset link has been sent." });
  } catch (error) {
    console.error("Forgot password error:", error);
    res.status(500).json({ message: "Failed to process request" });
  }
});
```

### POST /api/auth/reset-password

```typescript
app.post("/api/auth/reset-password", async (req, res) => {
  try {
    const { token, password } = req.body;
    if (!token || !password) {
      return res.status(400).json({ message: "Token and new password are required" });
    }

    const { validatePassword, hashPassword } = await import("./services/email-auth");
    const passwordCheck = validatePassword(password);
    if (!passwordCheck.valid) {
      return res.status(400).json({ message: passwordCheck.message });
    }

    const { pool } = await import("./db");
    const result = await pool.query(
      "SELECT id FROM users WHERE password_reset_token = $1 AND password_reset_expires > NOW()",
      [token]
    );

    if (result.rows.length === 0) {
      return res.status(400).json({ message: "Invalid or expired reset link. Please request a new one." });
    }

    const hashedPassword = await hashPassword(password);
    await pool.query(
      "UPDATE users SET password_hash = $1, password_reset_token = NULL, password_reset_expires = NULL, updated_at = NOW() WHERE id = $2",
      [hashedPassword, result.rows[0].id]
    );

    res.json({ message: "Password reset successfully. You can now log in." });
  } catch (error) {
    console.error("Reset password error:", error);
    res.status(500).json({ message: "Failed to reset password" });
  }
});
```

## 6. Session & Middleware

### Session Setup

```typescript
import session from "express-session";
import connectPg from "connect-pg-simple";
import passport from "passport";

function setupSession(app) {
  const sessionTtl = 7 * 24 * 60 * 60 * 1000; // 1 week
  const pgStore = connectPg(session);
  const sessionStore = new pgStore({
    conString: process.env.DATABASE_URL,
    createTableIfMissing: false,
    ttl: sessionTtl,
    tableName: "sessions",
  });

  app.use(session({
    secret: process.env.SESSION_SECRET,
    store: sessionStore,
    resave: false,
    saveUninitialized: false,
    cookie: {
      httpOnly: true,
      secure: true, // Set to false in development if not using HTTPS
      maxAge: sessionTtl,
    },
  }));

  app.use(passport.initialize());
  app.use(passport.session());

  passport.serializeUser((user, cb) => cb(null, user));
  passport.deserializeUser((user, cb) => cb(null, user));
}
```

### isAuthenticated Middleware

```typescript
const isAuthenticated = async (req, res, next) => {
  const user = req.user;

  if (!req.isAuthenticated() || !user) {
    return res.status(401).json({ message: "Unauthorized" });
  }

  // Email auth users don't have token expiry to check
  if (user.authProvider === "email") {
    return next();
  }

  // For OIDC/legacy users: check token expiry and refresh if needed
  // (remove this block if not supporting legacy auth)
  if (!user.expires_at) {
    return res.status(401).json({ message: "Unauthorized" });
  }
  const now = Math.floor(Date.now() / 1000);
  if (now <= user.expires_at) {
    return next();
  }
  // ... token refresh logic for legacy auth ...

  res.status(401).json({ message: "Unauthorized" });
};
```

### isAdmin Middleware

```typescript
const isAdmin = async (req, res, next) => {
  const userId = req.user?.claims?.sub;
  if (!userId) {
    return res.status(403).json({ message: "Forbidden" });
  }
  const user = await storage.getUser(userId);
  if (!user?.isAdmin) {
    return res.status(403).json({ message: "Forbidden" });
  }
  next();
};
```

## 8. Frontend Auth Hook

### `client/src/hooks/use-auth.ts`

```typescript
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import type { User } from "@shared/models/auth";

async function fetchUser(): Promise<User | null> {
  const response = await fetch("/api/auth/user", { credentials: "include" });
  if (response.status === 401) return null;
  if (!response.ok) throw new Error(`${response.status}: ${response.statusText}`);
  return response.json();
}

export function useAuth() {
  const queryClient = useQueryClient();
  const { data: user, isLoading } = useQuery<User | null>({
    queryKey: ["/api/auth/user"],
    queryFn: fetchUser,
    retry: false,
    staleTime: 1000 * 60 * 5,
  });

  const logoutMutation = useMutation({
    mutationFn: async () => {
      await fetch("/api/auth/logout", { method: "POST", credentials: "include" });
      window.location.href = "/";
    },
    onSuccess: () => {
      queryClient.setQueryData(["/api/auth/user"], null);
    },
  });

  return {
    user,
    isLoading,
    isAuthenticated: !!user,
    logout: logoutMutation.mutate,
    isLoggingOut: logoutMutation.isPending,
  };
}
```

**For FamilyRoots.family:** Since there's no legacy OIDC auth, the logout function simplifies to just `POST /api/auth/logout` + redirect. No need to check `authProvider`.

## 9. Admin Dashboard — User Management

### Backend: GET /api/admin/users

```typescript
app.get("/api/admin/users", isAuthenticated, isAdmin, async (req, res) => {
  try {
    const search = (req.query.search || "").trim().toLowerCase();
    const { pool } = await import("./db");

    let query = `
      SELECT
        u.id, u.email, u.first_name, u.last_name, u.created_at, u.is_admin, u.auth_provider,
        u.email_verified
      FROM users u
    `;
    const params = [];

    if (search) {
      params.push(`%${search}%`);
      query += ` WHERE (
        LOWER(u.first_name) LIKE $1 OR
        LOWER(u.last_name) LIKE $1 OR
        LOWER(u.email) LIKE $1
      )`;
    }

    query += ` ORDER BY u.created_at DESC`;
    const result = await pool.query(query, params);

    const users = result.rows.map((row) => ({
      id: row.id,
      email: row.email,
      firstName: row.first_name,
      lastName: row.last_name,
      createdAt: row.created_at,
      isAdmin: row.is_admin,
      emailVerified: row.email_verified,
      authProvider: row.auth_provider || "email",
    }));

    res.json(users);
  } catch (error) {
    console.error("Error fetching admin users:", error);
    res.status(500).json({ message: "Failed to fetch users" });
  }
});
```

**Note:** The PayGate version JOINs with `profiles` and `wallets` tables to show additional data (phone verification, wallet balance, profile visibility). Adapt the query to include whatever tables your app has.

## 10. Admin — Auth Migration Tool

This feature is for migrating users from a legacy auth provider (like Replit OIDC) to email/password auth.

### Backend: POST /api/admin/migrate-auth

```typescript
app.post("/api/admin/migrate-auth", isAuthenticated, isAdmin, async (req, res) => {
  try {
    const { userId } = req.body;
    if (!userId) {
      return res.status(400).json({ message: "User ID is required" });
    }

    const { pool } = await import("./db");
    const result = await pool.query(
      "SELECT id, email, first_name, auth_provider FROM users WHERE id = $1",
      [userId]
    );

    if (result.rows.length === 0) {
      return res.status(404).json({ message: "User not found" });
    }

    const user = result.rows[0];

    if (user.auth_provider === "email") {
      return res.status(400).json({ message: "User is already using email authentication" });
    }

    if (!user.email) {
      return res.status(400).json({ message: "User has no email address on file" });
    }

    const { generateToken } = await import("./services/email-auth");
    const resetToken = generateToken();
    const expires = new Date(Date.now() + 72 * 60 * 60 * 1000); // 72 hours for migration

    const appUrl = `${req.protocol}://${req.hostname}`;
    const resetUrl = `${appUrl}/reset-password/${resetToken}`;

    // IMPORTANT: Send email FIRST. Only update DB if email succeeds.
    try {
      await emailService.sendPasswordReset(user.email, user.first_name || "there", resetUrl);
    } catch (emailErr) {
      console.error("Failed to send migration email:", emailErr);
      return res.status(500).json({ message: "Failed to send password reset email. Migration aborted." });
    }

    // Email sent successfully — now update the user
    await pool.query(
      `UPDATE users SET auth_provider = 'email', email_verified = true, password_reset_token = $1, password_reset_expires = $2, updated_at = NOW() WHERE id = $3`,
      [resetToken, expires, userId]
    );

    res.json({ message: `Migrated ${user.email} to email auth. A password reset link has been sent.` });
  } catch (error) {
    console.error("Auth migration error:", error);
    res.status(500).json({ message: "Failed to migrate user" });
  }
});
```

**Key safety measures:**
1. Email is sent BEFORE the database is updated — if email fails, migration is aborted
2. `email_verified` is set to `true` for migrated users (they already verified via the legacy provider)
3. 72-hour token expiry (longer than normal 1-hour reset) gives migrated users time to act
4. The "Migrate" button is disabled for users without an email address

## 11. Migration from Replit Auth

If you're migrating from Replit OIDC (or any other OIDC provider), add this check in the OIDC verify callback to block migrated users:

```typescript
const verify = async (tokens, verified) => {
  const claims = tokens.claims();
  const existingUser = await authStorage.getUser(claims["sub"]);

  // Block users who have been migrated to email auth
  if (existingUser && existingUser.authProvider === "email") {
    return verified(null, false, {
      message: "This account has been migrated to email login."
    });
  }

  // Continue normal OIDC flow...
  const user = {};
  updateUserSession(user, tokens);
  await upsertUser(claims);
  verified(null, user);
};
```

And update the OIDC callback failure redirect:
```typescript
app.get("/api/callback", (req, res, next) => {
  passport.authenticate("oidc", {
    successReturnToOrRedirect: "/",
    failureRedirect: "/login?error=migrated", // Redirect to login with notice
  })(req, res, next);
});
```

**For FamilyRoots.family:** If you're starting fresh with no legacy auth, you can skip this section entirely. There's no OIDC to migrate from.

## 12. Security Considerations

1. **Password hashing:** scrypt with 16-byte random salt, 64-byte key length, timing-safe comparison
2. **Email case insensitivity:** All email comparisons use `LOWER()` in SQL
3. **Token security:** 32 random bytes (256 bits of entropy), hex-encoded
4. **Reset token expiry:** 1 hour for user-initiated resets, 72 hours for admin migrations
5. **Email enumeration prevention:** Forgot-password always returns the same success message regardless of whether the account exists
6. **Login enumeration:** Login DOES distinguish "no account" from "wrong password" — this is a UX tradeoff. The "no account" message helps guide users to sign up. If you prefer not to reveal this, return the same error for both.
7. **Verification enforcement:** Users cannot log in until their email is verified
8. **Migration atomicity:** Password reset email must send successfully before the auth provider is changed in the database
9. **Session security:** httpOnly, secure cookies, PostgreSQL-backed session store
10. **Admin protection:** All admin routes require both `isAuthenticated` and `isAdmin` middleware
